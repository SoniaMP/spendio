## Terms

| Term | Definition | Notes |
|------|------------|-------|
| Owner user | Account holder who creates sheets, categories and expenses. | Authenticates via Google OAuth or email + password. |
| Invited user | User granted access to a sheet owned by someone else. | Permission is `read` or `edit`. |
| Stub user | Account auto-created when a sheet is shared with an email that has no account yet. | Activated by setting a password via the standard reset flow. |
| Sheet | Container that groups expenses (e.g. "Personal", "Lisbon trip"). | One expense belongs to exactly one sheet. |
| Category | Per-user classification of an expense (name + color). | Name is unique per user; 8 defaults are created on signup. |
| Expense | A single spend line: amount, description, date, category, sheet. | `amount > 0`. |
| Expense payment | Partial reimbursement received from a third party for a given expense. | In design — see [`features/01.expenses.md`](features/01.expenses.md). 1:N to expense. |
| Effective amount | `expense.amount − SUM(expense_payments.amount)` for that expense. | Used by all monthly aggregates and charts when payments exist. |
| Sheet sharing | Grant of `read` or `edit` permission on a sheet to another user. | See [`features/02.sheets.md`](features/02.sheets.md). |
| Recurring expense | Template that auto-generates expense instances on a schedule. | Planned — see [`features/01.expenses.md`](features/01.expenses.md). |
| Dev login | Bypass authentication enabled by `VITE_AUTH_BYPASS=true`. | Local development only. |
